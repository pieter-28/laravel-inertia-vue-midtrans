# app-payment-gateway-midtrans Documentation

## 1. Project Overview

- **Project name**: app-payment-gateway-midtrans
- **Purpose**: Laravel + Vue Inertia app for product and transaction management with Midtrans payment integration.

## 1.1 Tech Stack

- **Backend**: Laravel 11 (PHP 8.2+)
- **Frontend**: Vue 3, Inertia.js, Vite
- **CSS**: Tailwind CSS
- **UI components**: shadcn/vue components (card, table, input, pagination)
- **Database**: MySQL (or compatible)
- **Authentication**: Laravel auth (Fortify or Breeze style)
- **Payment Gateway**: Midtrans (server + client key via `config/midtrans.php`)
- **Inertia adapter**: `@inertiajs/vue3`
- **Dev tools**: `npm`, `composer`, `php artisan`, `artisan migrate`, `artisan serve`

## 2. Key Features

- Authenticated product management (CRUD)
- Transaction listing with pagination
- Midtrans transaction status tracking
- Product checkout and payment endpoint
- Searchable transaction table with colored status badges

## 3. Folder Structure

- `app/Http/Controllers`
    - `ProductController`, `TransactionController`, `PaymentController`
- `app/Http/Resources`
    - `TransactionResource`, `ProductResource`, `CustomerResource`
- `app/Models`
    - `Product`, `Customer`, `Transaction`, `User`
- `resources/js/pages`
    - `Transaction/Index.vue`, `Product` pages, `auth` pages
- `resources/js/components/ui`
    - `card`, `table`, `input`, `pagination`, `button` components
- `routes`
    - `web.php`, `auth.php`, `settings.php`, `api.php`
- `config/midtrans.php`

## 4. Database Migrations

- `create_products_table`
- `create_customers_table`
- `create_transactions_table`
- `add_midtrans_fields_to_transactions_table`

## 5. Transaction Pagination (frontend)

- `resources/js/pages/Transaction/Index.vue`:
    - Table of transactions, status badge using `getStatusColor(status)`.
    - Search filter via `usePaginatedFilter` composable.
    - Pagination links from `props.transactions.links`.
    - `props.transactions.total > 0` condition to show footer.

## 6. Transaction Pagination (backend)

- `app/Http/Controllers/TransactionController.php`:
    - Query with `Transaction::with(['product','customer'])->latest()->paginate(10)->withQueryString()`.
    - Wrap data with `TransactionResource` and set collection on paginator.
    - Returns `transactions` paginator object to Inertia response.

## 7. Status badge mapping

- `pending`: `bg-yellow-100 text-yellow-800`
- `success` or `paid`: `bg-green-100 text-green-800`
- `failed`: `bg-red-100 text-red-800`
- default: `bg-gray-100 text-gray-800`

## 8. Setup and Run

1. `composer install`
2. `npm install`
3. Copy `.env.example` to `.env`
4. Configure DB and Midtrans keys
5. `php artisan key:generate`
6. `php artisan migrate --seed`
7. `npm run dev`
8. `php artisan serve`

## 9. Routes

- `/` -> Welcome
- `/dashboard`
- `/product` (resource)
- `/transaction` (resource)
- `/checkout/{id}`
- `/payment` (POST)

## 10. Notes

- Pagination links are generated by Laravel and consumed by Vue in `transactions.links`.
- `No transaction data available` is shown when empty.
- `transaction.status` is rendered as badge.
